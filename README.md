# 💬 ChatGo — Мессенджер на Go

> Разработка полноценного приложения для обмена сообщениями в реальном времени с поддержкой пользователей, авторизации и истории переписки.

---

## 🛠 Технологический стек

| Категория | Технология |
|---|---|
| 🔧 Контроль версий | Git |
| 🐹 Язык разработки | Go 1.23+ |
| 🌐 API | REST + WebSocket |
| 🗄 База данных | PostgreSQL 18 |
| 🔄 Миграции | golang-migrate |
| 🌍 Конфигурация | Переменные окружения (.env) |

| 🔐 Аутентификация | JWT + bcrypt |
| 📡 Реалтайм | WebSocket (gorilla/websocket) |

## ✨ Функциональные требования

### 1. 👤 Управление пользователями

> REST API для регистрации и аутентификации. JWT-токен выдается при успешном логине.

#### Доступные операции

| Операция | Эндпоинт | Описание |
|---|---|---|
| `POST` | `/api/register` | Регистрация нового пользователя |
| `POST` | `/api/login` | Логин, возвращает JWT-токен |

#### Валидация полей

**Никнейм (nickname)**
- ✅ Обязательное поле
- 📏 Минимум: **3 символа**
- 📏 Максимум: **15 символов**
- 🔒 Уникальное (не может совпадать с существующим)
- 🔤 Разрешены только латинские буквы (a–z, A–Z), цифры (0–9), символ подчёркивания (`_`) и дефис (`-`)

**Пароль (password)**
- ✅ Обязательное поле
- 📏 Минимум: **6 символов**
- 📏 Максимум: **10 символов**
- 🔒 Хранится в БД только в хешированном виде (bcrypt)

---

### 2. 💬 Обмен сообщениями

> WebSocket-соединение для обмена сообщениями в реальном времени. История сообщений сохраняется в БД.

#### Доступные операции (через WebSocket)

| Действие | Описание |
|---|---|
| Подключение | Установка WebSocket-соединения с передачей JWT-токена |
| Отправка сообщения | Клиент отправляет JSON с полями: `receiver_id`, `content` |
| Получение сообщения | Сервер мгновенно доставляет сообщение получателю (или уведомление) |
| Загрузка истории | При подключении клиент получает последние N сообщений из чата |

#### Валидация полей

**Сообщение (content)**
- ✅ Обязательное поле
- 📏 Минимум: **1 символ** (в рунах)
- 📏 Максимум: **1000 символов** (в рунах)
- 🔤 Допускаются любые печатные символы

**Получатель (receiver_id)**
- ✅ Обязательное поле
- 🔒 Должен существовать в системе (проверяется сервисным слоем)

**Отправитель (sender_id)**
- 🔒 Устанавливается автоматически из JWT-токена
